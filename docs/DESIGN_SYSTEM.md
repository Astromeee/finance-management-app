# Pocket Ledger Design System

## Theme

Pocket Ledger now uses a restrained premium fintech theme inspired by dark mobile finance apps. The UI is graphite-first, modular, compact, and intentionally low-glow.

## Color Palette

- Background: deep graphite / near-black `#121315` and `#0d0e10`.
- Card surface: elevated charcoal `#1b1c20`.
- Secondary surface: soft panel grey `#232428`.
- Borders: subtle translucent white/grey.
- Primary accent: neon-lime / chartreuse `#ddff45`, used sparingly for key actions and progress. Large lime blocks are avoided unless they are a small control.
- Positive: controlled green `#8be28f`.
- Negative: muted orange/red `#e98d67`.
- Informational: cool grey-blue `#aeb7c5`.
- Text: warm off-white primary with muted grey secondary.

## Typography

Headings are strong and compact. Large balances use confident sizing and tight tracking, while supporting labels remain small and muted. Mobile typography avoids oversized web-dashboard proportions.

## Spacing

Mobile uses 16px page padding, compact card rhythm, and safe bottom padding above the floating dock. Desktop uses wider spacing and stronger grid structure without stretching sections unnecessarily.

## Component Rules

- Cards are mostly flat charcoal panels with subtle borders and restrained shadows.
- Gradients are limited to the primary balance hero and are extremely subtle.
- Metric cards use small icon chips rather than colorful card-wide gradients.
- Progress bars use chartreuse for normal progress and muted orange/red for warnings.
- Transaction rows use compact icon blocks, title/subtitle hierarchy, and right-aligned amounts.
- Forms use dark inputs, subtle borders, clear labels, and chartreuse focus states.
- Account cards use a wallet-card style with tasteful per-account surfaces: sand for cash, graphite for HBL, muted green for Meezan, warm orange for JazzCash, and soft teal for Easypaisa.

## Mobile Navigation

The mobile nav is a slim floating dark dock with rounded corners, restrained shadow, icons only, muted inactive icons, and a white active icon on a raised dark cell. Content receives enough bottom padding so cards are never hidden behind it.

## Desktop Sidebar

The sidebar uses a flat near-black surface, chartreuse logo block, muted labels, and an active state with a small accent rail instead of a loud gradient.

## Gradients And Shadows

Gradients are intentionally minimal. Heavy cyan/purple glow and decorative glassmorphism are avoided. Shadows are soft, dark, and used only to create hierarchy.
